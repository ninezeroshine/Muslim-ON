**Версия ТЗ:** 1.0  
**Дата:** 06.04.2026  
**Заказчик:** [Вы]  
**Цель:** Создать высококачественное, стабильное, современное Windows-приложение для автоматического напоминания о намазе и выключения ПК по выбранным молитвам с возможностью дальнейшего расширения исламским функционалом.

### 1. Общее описание и цели приложения
Приложение синхронизируется с точными временами 5 намазов (Фаджр, Зухр, Аср, Магриб, Иша) по выбранному мазхабу/методу расчёта и локации (пример: Казань, Татарстан).  
Пользователь настраивает:
- Локацию и метод расчёта (мазхаб).
- Для каких намазов включён режим «выключения ПК».
- Напоминание за 15 минут до намаза (toast-уведомление + звук/адхан по желанию).
- Если напоминание проигнорировано → через 15 минут после начала намаза → автоматическое выключение ПК.

**Ключевые принципы:**
- Максимальная стабильность и native-вид под Windows 11/10.
- Полностью оффлайн-работа (расчёт времён локально) + опциональная синхронизация с API для актуальности.
- Модульная, масштабируемая архитектура (чтобы потом легко добавить: чтение Корана, дуа, hijri-календарь, qibla-компас, трекер поста, уведомления о пятнице и т.д.).
- Минимальный размер, низкое потребление ресурсов, запуск в фоне (tray-иконка).

### 2. Функциональные требования (MVP + расширяемость)

**Core-функционал (обязательно в первой версии):**
1. **Настройка локации и метода расчёта**
   - Выбор города (предустановленный список + поиск + ручной ввод координат).
   - Автоматическое определение локации через Windows Geolocation API (с согласия пользователя).
   - Выбор метода расчёта (MWL, Egyptian, Karachi, Umm al-Qura, ISNA, Makkah и др.) + отдельно метод Асра (Shafi/Hanafi).
   - Учёт часового пояса, DST, high-latitude rules.

2. **Ежедневный расчёт времён намазов**
   - Локальный расчёт на сегодня/завтра/месяц вперёд.
   - Кэширование расчётов (SQLite).
   - Автоматическое обновление расчётов в полночь или при смене даты.

3. **Настройка правил выключения**
   - Чекбоксы для каждого из 5 намазов (можно выбрать Зухр, Аср, Магриб и т.д.).
   - Глобальные настройки: «напоминать за 15 мин», «выключать через 15 мин после начала».

4. **Система напоминаний и действий**
   - За 15 минут до намаза → toast-уведомление (Fluent Design) + опционально звук адхана.
   - Кнопки в уведомлении: «Я прочитал» / «Отложить на 5 мин» / «Выключить ПК сейчас».
   - Если игнор → через 15 мин после начала намаза → мягкое предупреждение → выключение ПК (`shutdown /s /t 0` или Win32 `InitiateSystemShutdownEx`).
   - Логирование действий (для отладки и будущего аналитики).

5. **Tray-иконка и фоновая работа**
   - Постоянная иконка в системном трее (текущий следующий намаз + обратный отсчёт).
   - Двойной клик → открытие окна настроек.
   - Запуск при старте Windows (Registry/Task Scheduler).

**Модульность и будущее расширение (архитектура должна это поддерживать из коробки):**
- Возможность добавлять новые «модули» (Quran Reader, Dua of the Day, Hijri Calendar, Fasting Tracker, Qibla Compass и т.д.) без переписывания ядра.
- Каждый модуль — отдельный feature с собственными сервисами, UI-страницами и настройками.

### 3. Нефункциональные требования
- **Производительность:** Запуск < 1 сек, потребление RAM < 80 МБ в фоне.
- **Стабильность:** 100% offline после первой настройки, graceful degradation при проблемах с API.
- **UI/UX:** Полностью Fluent Design (WinUI 3), тёмная/светлая тема, поддержка Windows 10 1809+ и Windows 11.
- **Локализация:** Русский + английский + арабский (минимум).
- **Доступность:** Поддержка screen readers, высококонтрастный режим.
- **Безопасность:** Нет отправки данных пользователя (кроме опционального API-запроса). Все данные локальные.
- **Размер дистрибутива:** < 30 МБ (single .exe или MSIX).

WinUI 3 + .NET 10 — это **официальный путь Microsoft** для качественных Windows-приложений в 2026 году. Никаких костылей, максимальная интеграция с системой, лучший DX (Visual Studio + Hot Reload + Designer).

### 5. Архитектура приложения (модульная и масштабируемая)

**Структура решения (рекомендуемая):**
```
PrayerShutdown.Solution
├── PrayerShutdown.Core          ← Domain + Application (модели, сервисы, interfaces)
├── PrayerShutdown.Features      ← Feature folders (Prayer, Settings, Notifications и т.д.)
├── PrayerShutdown.Services      ← Инфраструктура (PrayerCalculator, ShutdownService, Storage и т.д.)
├── PrayerShutdown.UI            ← WinUI 3 проект (Views, ViewModels, Resources)
├── PrayerShutdown.Tests
└── PrayerShutdown.Common        ← Общие утилиты
```

- **Vertical Slice Architecture** — каждый feature (например, PrayerModule) содержит свои команды/запросы/handlers.
- **Dependency Inversion** — все модули зависят от абстракций.
- В будущем новый модуль (например, QuranModule) добавляется как отдельный проект/папка без изменения ядра.
